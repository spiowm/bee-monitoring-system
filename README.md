# 🐝 Bee Monitoring System (BuzzTrack)

Комплексна AI-система для комп'ютерного зору та аналізу поведінки бджіл на прилітній дошці вулика. Проєкт створено для автоматизованого моніторингу активності бджолиних сімей, підрахунку трафіку (вліт/виліт) та розпізнавання патернів поведінки.

## 🏗 Архітектура проєкту

Система побудована як монорепозиторій і складається з трьох повністю незалежних компонентів:

1. **`research/` — ML & MLOps середовище**
   - Навчання моделей YOLO-pose для визначення пози бджіл (2 keypoints: голова, жало).
   - Тренування детектора посадкової дошки вулика (ramp detector).
   - Інфраструктура для автоматичного підбору гіперпараметрів (Optuna), керування конфігами (Hydra) та ML-трекінгу (MLflow + DagsHub).
   - Детальна інструкція: [research/README.md](research/README.md).

2. **`backend/` — REST API сервер (FastAPI)**
   - Процесує завантажені відео у фонових задачах (BackgroundTasks).
   - Pipeline обробки (відео → YOLO → ByteTrack → Аналіз напрямку → Аналіз поведінки → Збереження).
   - Два режими підрахунку перетину: A (тільки за траєкторією) та B (верифікація напрямку руху за допомогою вектору пози бджоли).
   - Зберігає всі події перетину локацій та аналітику в **MongoDB**.
   - Підготовка H.264 відео з накладеними боксами та статистикою для веб-відображення.

3. **`frontend/` — Веб-інтерфейс (React + TypeScript + Vite)**
   - Дашборд для завантаження відео з вуликів.
   - Графічний інтерфейс для налаштування гіперпараметрів трекінгу.
   - Живе відображення статистики процесингу (Live Stats) через polling.
   - Аналітика після обробки: порівняння метрик методів підрахунку, загальний трафік, програвач для перегляду анотованих відео.

## 🚀 Швидкий старт

Для роботи системи потрібна запущена **MongoDB** (локально, або у хмарі напр. MongoDB Atlas).

### 1. Запуск Backend (FastAPI)

```bash
cd backend
# Встановлення залежностей через uv
uv sync

# Створення .env файлу
echo "MONGO_URI=mongodb://localhost:27017" > .env

# Запуск сервера
uv run uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
API буде доступне за адресою: `http://localhost:8000/docs`

### 2. Запуск Frontend (React)

```bash
cd frontend
npm install
npm run dev
```
Відкрийте браузер за адресою: `http://localhost:5173`

## 🧠 Ключові алгоритми та евристики

### Верифікація за позою (Approach B)
Звичайний трекінг не завжди ідеальний. Інколи бджола може задкувати, або трекер "губить" আইডি і перепризначає його іншій бджолі, що летить назустріч. Щоб розв'язати це, система використовує YOLO-pose для визначення двох ключових точок (голова, жало). 
Якщо напрямок вектору "жало → голова" розходиться з вектором руху треку більше ніж на `angle_threshold_deg` — такий перетин відхиляється.

### Розпізнавання поведінки (Behavior Baseline)
Наразі система використовує просту базову евристичну модель (baseline) для розпізнавання поведінки бджіл, яка базується на метриках швидкості та площі переміщення:
- **Foraging:** висока швидкість руху в кадрі.
- **Fanning (вентиляція):** дуже повільний рух, бджола тривалий час знаходиться на одній позиції.
- **Guarding (охорона):** рух переважно вздовж входу (горизонтальний spread > вертикального).
- **Washboarding:** фоновий патерн для інших типів низькоінтенсивних пересувань.
*(Зауваження: це базові пороги для Proof-of-Concept, які можна замінити на більш складну Time-Series ML модель у майбутньому).*

## 🛠 Технологічний стек

- **ML & CV:** PyTorch, Ultralytics (YOLO11), Supervision, OpenCV.
- **MLOps:** Hydra, Optuna, MLFlow, DagsHub, Kaggle API.
- **Backend:** Python 3.12+, FastAPI, Motor (Async MongoDB), Pydantic.
- **Frontend:** React, TypeScript, Vite, TailwindCSS (via custom index.css variables), Axios.
- **Package Manager:** `uv` (швидкий менеджер пакетів для Python від Astral).
